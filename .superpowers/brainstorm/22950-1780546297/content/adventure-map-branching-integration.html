第二版方向：大学生能力地图，不做竖直闯关

你说的两个问题我都改了：路径不再垂直向上，而是横向展开、多分支；节点也不再是“数学/英语/阅读”这种窄学科，而是大学生探索专业和职业方向时真正要积累的能力、证据和作品。

学习探索地图 按画像、方向、任务和证据实时生成
学生当前位置 从当前画像和 workspace 进度计算，不固定站在起点。 头像随任务完成移动
专业底座 当前推荐 来自 recommended_knowledge + 低分画像维度
核心能力 挑战点 来自 match_report 的优先差距
项目作品 实践任务 来自 exploration_tasks / mini_project
资源证据 已完成 来自 workspace.resources completed
方向探索 画像更新 来自 career_directions / profile_versions
实习作品集 待解锁 等短期证据足够再推进
沟通协作 软能力 来自 12 维画像与教练建议
接入原则 底图是静态高质量资产；节点、文案、状态、头像位置全部由现有 plan/workspace 派生。这样看起来像冒险地图，行为上仍然是你的真实探索系统。

为什么这比一条学习路径更适合大学生

  • 大学生不是只按科目闯关，而是在专业基础、方向判断、项目实践、资源证据、实习准备之间来回探索。
  • 地图应展示“当前为什么推荐这里”，而不是假装所有人都沿同一条路走。
  • 节点可以有分支和回路：补基础、做项目、更新画像、换方向，都应该能反映在地图上。

怎么接入现有数据并实时显示

plan.knowledge_map生成专业底座、核心能力、方向探索、实践任务这些地图节点。
plan.recommended_knowledge决定当前推荐节点，并把 reason 传给生成模块。
match_reports把优先差距变成挑战点，比如核心能力、沟通协作、作品集。
workspace.phases任务 done 后，节点变已完成，头像移动到最新完成/当前任务附近。
workspace.resources资源 opened/completed 后，资源证据节点实时变状态。
profile_versions画像被更新后，相关节点高亮“画像变化”。
