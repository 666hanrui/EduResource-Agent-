i

共享视觉系统重构

主页和老师端共用 bkhlbb 的版式骨架：hero、spread、toc、ledger、section head、footer。老师端只保留业务交互，不再保留 mesh 外观。

  • 效果最统一，也最接近你给的 HTML
  • 改动稍大，但能把“毁掉”的风格彻底拉回来
ii

局部 CSS 覆盖修复

继续沿用现在的 React 结构，只把颜色、字体、圆角、阴影覆盖掉。速度快，但底层还是卡片/终端/面板混合感。

  • 风险小，速度最快
  • 容易继续出现“不像参考页”的割裂
iii

主页参考风，老师端后台风

主页完全像参考页，老师端保留科技工作台。逻辑上清楚，但整体项目会像两个产品拼起来。

  • 老师端功能感最强
  • 会继续破坏整体一致性
