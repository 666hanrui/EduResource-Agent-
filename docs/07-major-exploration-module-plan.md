# 专业探索模块改造计划

## 背景

`career-planning-agent/feature-agentic` 的核心链路是：

1. 上传简历或补充材料。
2. 生成学生就业能力 12 维画像。
3. 基于画像做职业/岗位匹配。
4. 收藏目标岗位。
5. 生成蜗牛学习路径、复盘和个人成长报告。

EduResource-Agent 面向的是刚入学或低年级大学生。这个阶段的学生通常没有成熟简历，也没有稳定目标岗位。因此不能沿用“简历 -> 岗位匹配”的前置假设，而应该改成“专业 -> 广度探索 -> 兴趣收敛 -> 职业方向假设 -> 学习资源生成”。

## 改造目标

把 `feature-agentic` 中可复用的 12 维画像、职业方向匹配、学习路径、复盘、成长报告能力，改造成 EduResource-Agent 的一个独立模块：专业探索模块。

新的主链路：

```text
选择专业/年级/基础水平
  -> 12 维探索型初始画像
  -> 专业知识广度地图
  -> 探索任务和兴趣反馈
  -> 候选职业方向
  -> 个性化资源生成
  -> 学习反馈更新画像
```

## 概念映射

| feature-agentic 概念 | EduResource-Agent 概念 | 改造方式 |
|---|---|---|
| 简历解析 | 探索型画像初始化 | 输入从简历文件改为专业、年级、基础自评、兴趣问卷、诊断题 |
| 目标岗位必填 | 候选职业方向后置生成 | 入门阶段不要求目标岗位，系统推荐 3-5 个可探索方向 |
| 就业能力 12 维画像 | 学习与职业探索 12 维画像 | 保留 12 维 key，调整文案和证据源 |
| 岗位匹配 | 职业方向探索推荐 | 从“找最匹配岗位”改为“生成可验证方向假设” |
| 收藏岗位 favorite_id | 收藏探索方向 direction_id | 后续学习路径围绕方向或知识域展开 |
| 蜗牛学习路径 | 专业探索学习路径 | 保留短/中/长期计划、资源推荐、里程碑、复盘 |
| 个人职业成长报告 | 专业探索成长报告 | 报告内容从求职材料改为探索记录、兴趣证据、阶段能力变化 |
| ResumeCoach | ExplorationCoach | 改成专业探索教练，避免简历优化语义 |

## 保留能力

优先保留以下功能思想和数据结构：

- 12 维画像字段和分组。
- 能力雷达、差距分析、关键字编辑。
- NDJSON / SSE 异步进度流。
- 职业/岗位要求画像与学生画像对比的思想。
- 收藏目标后生成学习路径。
- 短期、中期、长期三阶段规划。
- 学习资源卡片、打开外链、打卡。
- 周/月复盘和证据沉淀。
- 成长报告生成、编辑、导出。
- AI 教练多角色协作。

## 必须修改的前置假设

- `target_job_title` 不能作为注册/引导必填项。
- 简历上传不能作为默认入口。
- “工作经验”维度不能只理解为实习经历，应覆盖课程项目、社团实践、实验课、竞赛、个人作品、探索任务。
- “学历要求”对学生画像应表达当前学历和年级，不应表达岗位筛选门槛。
- 职业匹配不能直接给出确定结论，应输出“候选方向 + 为什么值得探索 + 验证任务”。
- 学习路径的起点不应依赖已收藏岗位，可以依赖专业知识域或探索方向。

## 12 维画像适配

保留原始英文 key，降低迁移成本：

| Key | 新语义 |
|---|---|
| `professional_skills` | 当前已接触的专业技能、工具、方法 |
| `professional_background` | 专业、课程、知识基础和学科兴趣 |
| `education_requirement` | 当前学历、年级、课程阶段 |
| `teamwork` | 小组作业、社团、课程协作倾向 |
| `stress_adaptability` | 面对考试、项目、陌生知识的适应方式 |
| `communication` | 表达、汇报、写作、讨论偏好 |
| `work_experience` | 课程项目、实验、竞赛、社团、实习和个人作品 |
| `documentation_awareness` | 笔记、报告、实验记录和文档习惯 |
| `responsibility` | 任务推进、按时交付、自我管理 |
| `learning_ability` | 自学、迁移、复盘、主动补齐能力 |
| `problem_solving` | 拆解问题、调试、分析、解决问题 |
| `other_special` | 证书、语言、兴趣、家庭/地域约束、其他补充 |

## 后端分期

### Phase 1：本地规则版专业探索模块

目标：不依赖数据库、不依赖外部 Dify，先在当前轻量项目内打通完整演示。

新增：

- `backend/app/schemas/exploration.py`
- `backend/app/services/major_exploration.py`
- `/api/exploration/plan`

输入：

- 学生 ID
- 专业
- 年级
- 学历
- 基础水平
- 兴趣关键词
- 每周可投入时间

输出：

- 12 维探索画像
- 12 维雷达和分组
- 专业知识广度地图
- 探索任务
- 候选职业方向
- 三阶段学习路径
- 可被 Resource 7-Agent 继续使用的推荐知识点

### Phase 2：接入现有 7-Agent 资源生成

把用户选中的 `recommended_knowledge` 送入 `/api/generate`：

- `knowledge_id`
- `knowledge_name`
- `prior_profile`

这样专业探索模块负责“找方向和知识点”，现有 7-Agent 负责“生成学习资源”。

### Phase 3：持久化探索过程

新增内存/SQLite 存储：

- 探索会话
- 方向收藏
- 任务反馈
- 画像版本
- 学习路径状态

### Phase 4：迁移 feature-agentic 的学习路径与报告能力

把下列能力移植为轻量版：

- 收藏方向替代收藏岗位。
- 短/中/长期路径工作区。
- 学习资源打卡。
- 周/月复盘。
- 成长报告生成和编辑。

### Phase 5：升级到真实职业/专业知识库

引入结构化数据：

- 专业基础课程图谱。
- 方向与职业族群映射。
- 岗位画像数据。
- 学习资源库。
- 向量检索或图谱检索。

## 前端分期

### Phase 1：专业探索工作台

新增 `MajorExplorationPanel`：

- 基础信息输入。
- 生成探索计划。
- 左侧 12 维画像。
- 中间专业知识地图。
- 右侧职业方向和学习路径。
- 点击知识点可回填到现有资源生成表单。

### Phase 2：能力画像编辑

复用 feature-agentic 的关键字编辑思想：

- 每个维度显示关键词。
- 支持增删关键词。
- 保存后刷新方向推荐。

### Phase 3：方向收藏与路径工作区

- 收藏候选方向。
- 生成三阶段路径。
- 打卡探索任务。

### Phase 4：成长报告

- 汇总探索记录、兴趣变化、能力变化。
- 支持导出。

## 当前项目接入策略

当前 EduResource-Agent 还不是完整多页面应用，因此先采用低风险接入：

- 后端直接挂到现有 `build_router(ctx)`。
- 前端 `App.tsx` 增加 tab，保留原资源生成演示页。
- 新模块不破坏已有 `/api/generate`、`AgentTracePanel`、`ResultsPanel`。
- 专业知识图谱与职业画像先落到 `backend/app/data/major_catalog.py`，后续可替换为数据库或运营后台。
- 收藏方向、探索工作区、任务、复盘、资源状态、报告草稿和画像版本落到 SQLite：`backend/.data/exploration_store.sqlite3`。
- 可通过 `EDU_EXPLORATION_DB_PATH` 指定生产部署的数据文件路径。

## 验收标准

Phase 1 完成时应满足：

- `POST /api/exploration/plan` 返回 12 维画像、知识地图、方向推荐和三阶段学习路径。
- 前端可以填写专业信息并看到专业探索结果。
- 前端可以从探索结果选择一个知识点，回填并启动现有 `/api/generate`。
- 后端测试覆盖画像字段、知识地图、方向推荐和学习路径。
- `pytest`、`npm run typecheck`、`npm run build` 通过。

## 当前落地状态

已完成：

- 规则版 `POST /api/exploration/plan`。
- 前端“专业探索”工作台。
- 静态专业 catalog：计算机/电子通信/经管/默认专业，包含基础课、核心课、探索方向、实践任务和职业画像。
- 职业方向推荐带岗位画像要求：核心技能、典型任务、12 维权重和证据建议。
- 推荐知识点回填到现有 7-Agent 资源生成链路。
- 方向收藏与路径工作区的 SQLite 持久化实现。
- 工作区任务打卡。
- 学习资源卡片、外链打开和完成打卡。
- 周/月复盘记录。
- 画像关键词编辑和版本化。
- 规则版 ExplorationCoach，可按鼓励/诊断/挑战语气给下一步探索建议。
- Markdown 成长报告生成、编辑稿保存、Markdown/HTML 导出。
