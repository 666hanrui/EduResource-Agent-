# 02 · 项目创新点（扩写版）

> 第十四章 · 用工程化的可见性与可解释性，建立差异化

---

本系统的创新不在"用大模型做了一个教育产品"，而在 **让多智能体协作过程对学生和评委都可见、可解释、可追溯**。围绕这一核心思想，重点打造两个差异化能力，并通过四项工程机制将"个性化"从口号转化为可被验证的现场效果。

---

## 14.1 杀手锏一：Agent 协作时序可视化

### 问题动机

市面上多数"AI 教育助手"对外只暴露一个对话框，背后无论是单 LLM 还是多 Agent，对用户而言完全是黑盒。这导致两个后果：用户无法判断系统在做什么、为什么慢；评委无法判断作品是真"多智能体"还是"一个 Prompt 包了多个角色"。

### 解决方案

我们把 Agent 调度核心的内部状态以 SSE 事件流形式暴露给前端，前端在"资源工厂"页右侧渲染**实时时序面板**。每个 Agent 占一行，状态用四种视觉语言区分：等待（灰）、流式生成（蓝色呼吸）、完成（绿色实条）、失败（红色）。横向时间条直观体现并行度——`DocumentAgent` 与 `ExerciseAgent` 是并行的，`CodeAgent` 等待文档完成后启动，整个 DAG 一目了然。

### 技术实现

后端的每个 Agent 在生命周期关键节点（start / step / done / error）向调度核心发送事件；调度核心通过 NDJSON 格式的 SSE 流推给前端；前端维护一个 `Map<agentName, AgentState>`，按事件 reducer 式更新。Prompt 版本号、模型名、token 消耗、引用资料等元信息都附在事件中，鼠标悬停即可查看。这意味着：**评委 5 秒就能确认"这是真的多 Agent 系统"**。

---

## 14.2 杀手锏二：资源生成可解释性溯源

### 问题动机

"个性化推荐"四个字在大模型时代极易沦为话术。学生看到的是一份文档、一道题，根本无法判断它和"普通 ChatGPT 答的"有什么区别。如果系统不能回答"为什么是这份"，那"个性化"就只是 Prompt 里塞了一句"基于学生画像"。

### 解决方案

每张资源卡片下方设置 **固定四段式溯源区**：

1. **画像匹配** —— 列出本资源响应了画像哪些维度（学习风格 / 资源偏好等），并标明影响了生成的哪些参数；
2. **短板对应** —— 列出本资源针对的薄弱知识点，引用最近 N 次错题作为依据；
3. **难度自适应** —— 显示当前掌握度、原默认难度、最终采用难度，让"自适应"看得见；
4. **生成参数追溯** —— 标注 Agent 名 / Prompt 版本 / 模型 / 引用资料及向量相似度。

### 技术实现

每个生成 Agent 输出的 Pydantic Schema 中强制包含 `rationale` 字段，结构化记录上述四项。前端不调用 LLM 二次解释——溯源信息**全部来自生成时的真实参数**，可信、可复现。学生可对资源做👍/👎反馈，反馈信号回流到 ProfileAgent，下一轮推送会自动调整。这样推荐过程不再是"信任大模型"，而是"信任一条可审计的数据链"。

---

## 14.3 工程机制 · Schema 强约束的 Agent 输出

所有 Agent 输出使用 Pydantic Schema 约束，包括 `rationale`、引用资料、参数选择等元信息。LLM 生成失败或字段缺失时自动重试，三次失败后降级为模板兜底。这一机制让"演示翻车"风险可控，是 35 分创新分背后的工程兜底。

---

## 14.4 工程机制 · 跨模态一致性

针对同一知识点（如"链表插入"），文档、题目、代码、动画由 PlannerAgent 在同一上下文中下发任务并共享同一份"知识点解构 JSON"作为锚点。这保证：文档里讲到的"prev/next 修改顺序"，会出现在题目错误选项、代码案例的注释、动画的高亮帧中。**评委一眼能看出资源不是 4 个独立 LLM 调用拼出来的**，而是一个连贯的学习闭环。

---

## 14.5 工程机制 · 双轨多模态策略

主流程使用前端 SVG/Canvas 实现交互式动画（步骤可暂停、可回放、可联动文档高亮），保证演示稳定、零调用成本。同时在演示视频中嵌入 2 段 SeeDance 预生成的教学短片，覆盖赛题点名的多模态生成工具。这一策略让稳定性和"政策性加分项"兼得。

---

## 14.6 工程机制 · 闭环现场可演示

整个系统设计的最终目标是 **现场 2 分钟演示出闭环**：学生答错 → EvaluationAgent 实时分析 → 画像 `weakness` 字段刷新 → 路径中下一题难度自动下调 → 重新推送对应动画 → 学生答对 → 掌握度回升。所有数据预置真实、调用真实，不靠剪辑。这是把"个性化"从 PPT 名词变成评委亲眼所见的最直接武器。

---

## 14.7 创新点小结

|  | 创新方向 | 评委可验证标志 |
|---|---|---|
| 杀手锏一 | Agent 时序可视化 | 5 秒看到多 Agent 并行 |
| 杀手锏二 | 资源溯源四段式 | 点开任何资源都能看到"为什么是它" |
| 工程① | Schema 强约束 | 演示不翻车 |
| 工程② | 跨模态一致性 | 文档/题/码/动画讲同一件事 |
| 工程③ | 双轨多模态 | 既稳定又覆盖 SeeDance |
| 工程④ | 闭环现场演示 | 2 分钟看到画像在动 |

这套创新的共同特征是：**不依赖更强的模型，而依赖工程化的可见性与可解释性**。这让我们的方案在大量"调 LLM 包装一下"的参赛作品中具备清晰的辨识度。
