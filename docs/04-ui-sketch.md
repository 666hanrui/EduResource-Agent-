### 视觉关键

- 时间条用渐变色（已完成绿、流式蓝呼吸、等待灰、失败红）
- 每个 Agent 一行，**鼠标悬停**展开 Prompt 版本号 / 模型名 / token 用量
- 点击"展开调用日志"切换为甘特图视图（横轴时间，纵轴 Agent）
- 演示视频 0:30 一定要给这个面板一个 5 秒特写

### 状态机

```
WAITING  ──→  RUNNING (start 事件)
RUNNING  ──→  STREAMING (首条 delta)
STREAMING ─→  DONE (done 事件)
RUNNING  ──→  ERROR (error 事件，可重试)
```

## 杀手锏二：资源可解释性溯源卡片

每张生成的资源卡片**底部都有一个折叠区**："为什么推荐给你"。

### 收起态

```
┌──────────────────────────────────────────────────────────────┐
│  📄 链表插入操作详解              难度 ★★☆☆☆   讲解文档    │
│                                                              │
│  本节聚焦单链表的中间插入操作。我们将先理解指针的修改顺序…   │
│  [打开全文]                                                  │
│                                                              │
│  ─────────────────────────────────────────────────────────  │
│  💡 为什么推荐给你？                              [展开 ▼]  │
└──────────────────────────────────────────────────────────────┘
```

### 展开态

```
┌──────────────────────────────────────────────────────────────┐
│  💡 为什么推荐给你？                              [收起 ▲]  │
│                                                              │
│  ① 画像匹配                                                  │
│     · 学习风格：图解 + 分步骤讲解  ──→ 文档含 4 张步骤图    │
│     · 资源偏好：思维导图、动画     ──→ 文档已绑定动画 #A03 │
│                                                              │
│  ② 短板对应                                                  │
│     · 历史易错点：「指针修改顺序」（最近 3 题错 2）          │
│     ──→ 文档第 3 节专门讲解 prev/next 的安全顺序             │
│                                                              │
│  ③ 难度自适应                                                │
│     · 当前掌握度：链表 0.42（薄弱）                          │
│     ──→ 难度从默认 3 自动降为 2                              │
│                                                              │
│  ④ 生成参数追溯                                              │
│     · DocumentAgent v1.3 · 星火 X2 · prompt#doc-step-img    │
│     · 引用资料：教材 P127-130（向量召回 0.89）               │
│                                                              │
│  [👍 推荐准确]   [👎 不太合适]   [查看完整溯源链]           │
└──────────────────────────────────────────────────────────────┘
```

### 关键设计

- 4 段固定模板（画像匹配 / 短板对应 / 难度自适应 / 生成参数）
- 用户点👍👎会作为**反馈信号**回流到 ProfileAgent，是闭环的一部分
- "查看完整溯源链"打开知识图谱视图：高亮"画像维度 → 短板节点 → 资源生成路径"

### 数据来源

每个生成 Agent 输出 schema 中强制包含：

```python
class Rationale(BaseModel):
    matched_profile: list[str]          # ① 画像匹配
    addressed_weakness: list[str]       # ② 短板对应
    difficulty_adjusted_from: int
    difficulty_used: int                # ③ 难度自适应
    agent_name: str
    prompt_version: str
    model_name: str
    cited_sources: list[CitedSource]    # ④ 生成参数追溯
```

前端 **不调用 LLM 二次解释**，直接渲染这条结构化数据。

